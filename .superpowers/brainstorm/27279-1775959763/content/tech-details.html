🎥 编码参数 — 消除色带

remotion.config.ts 关键配置
// 分辨率
width: 1080
height: 1920
fps: 30 ← 从 24 提升到 30，动画更流畅
// 编码 — 10-bit H.265 消除色带
codec: "h265"
pixelFormat: "yuv420p10le" ← 10-bit 色深
crf: 18 ← 高质量 (越低越好，18 是视觉无损)
// 抖音兼容性备选 (如遇兼容问题)
// codec: "h264"
// pixelFormat: "yuv420p"
// crf: 15 ← 高码率 + film grain 弥补 8-bit 色带
// 音频
audioBitrate: "192k" ← 从 128k 提升

🔊 音频同步方案

TTS timestamps → Remotion 帧映射
// 1. Python TTS 生成 timestamps.json (不变)
[{text:"微软", offset_ms:350, duration_ms:400}, ...]
// 2. Python 计算每段 TTS 时长 → 段时间分配
sections_timing.json: [{type:"title", start_s:0, end_s:5.2}, ...]
// 3. Remotion 读取 timing → 每个 scene 知道自己的帧范围
const frame = useCurrentFrame()
const {fps} = useVideoConfig()
const progress = frame / (scene.durationFrames)
// 4. 字幕：Remotion 内置渲染，不再用 ASS
timestamps → Subtitle 组件 → 逐字高亮
// 5. 音频：直接引入 mp3 作为 Remotion Audio
<Audio src={audioUrl} />

🔄 Python → Remotion 调用流程

composer.py 中的新 _compose_short_v3()
# 1. 准备输入 JSON
input_data = {sections, timestamps, audio_path, seed}
write_json(tmp / "input.json", input_data)
# 2. 调用 Remotion CLI 渲染
npx remotion render ShortVideo \
--props=tmp/input.json \
--output=output.mp4 \
--codec=h265 \
--crf=18
# 3. 返回结果 (和现有接口一致)
return {"video_path": ..., "duration_seconds": ...}
